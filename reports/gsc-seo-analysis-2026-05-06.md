# Calkilo Google Search Console SEO Analysis

Analysis date: 2026-05-06

Search Console filters: Web search, last 3 months  
Observed date range in chart export: 2026-02-04 to 2026-05-03

## Executive Summary

Calkilo is starting to get meaningful non-brand search visibility. Total exported chart performance is 89 clicks, 2,317 impressions, 3.84% CTR, and weighted average position 8.04.

The strongest growth happened in late April. The week starting 2026-04-28 produced 790 impressions and 35 clicks, compared with 58 impressions and 2 clicks in the first exported week. This is a visibility breakout, but not yet a CTR breakout.

The main opportunity is not broad technical SEO. The site already has canonical tags, indexable static pages, hreflang on localized homepages, JSON-LD, a robots.txt file, and a sitemap. The opportunity is query-to-page fit: Persian and Italian searchers are seeing Calkilo near page one, but the site does not yet have enough language-specific, intent-specific landing pages for the queries that are already appearing.

## Data Caveats

## Priority Recommendations

### 1. Build Persian intent pages, not just a Persian homepage

Create dedicated Persian resource pages for:

- `/fa/photo-calorie-calculator/`
- `/fa/ai-calorie-tracker/`
- optionally `/fa/macro-tracker/`

Use the actual query language in titles, H1s, meta descriptions, internal links, and visible body copy. The homepage already says "کالری شمار رایگان با هوش مصنوعی", but the export shows two distinct intents: "AI calorie counter" and "photo calorie counter". Separate pages should let Google match each query to a more specific canonical URL.

Suggested Persian title patterns:

- `کالری شمار با عکس رایگان | محاسبه کالری غذا با هوش مصنوعی`
- `کالری شمار هوش مصنوعی رایگان | Calkilo`
- `اسکن کالری غذا با عکس | کالکیلو`

Each page should include practical sections: how photo calorie estimation works, what the user can edit, when estimates may need correction, macro tracking, privacy, app availability, and short FAQs.

### 2. Improve Italian matching for `calcolo calorie ai`

The Italian homepage is getting impressions but almost no clicks. The exact query `calcolo calorie ai` has 37 impressions, average position 8.59, and zero clicks.

Recommended Italian copy angles:

- `Calcolo calorie AI da foto`
- `contacalorie con foto`
- `app per calorie e macro`
- `analisi alimentare con intelligenza artificiale`

Create an Italian guide page or strengthen `/it/` above the fold so the title, H1, and first paragraph directly match "AI calorie calculation from food photos".

### 3. Rework CTR snippets where rankings are already near page one

Google's title and snippet guidance emphasizes clear, concise titles and unique page descriptions. Apply this first to `/fa/`, `/it/`, `/photo-calorie-calculator/`, and `/features/`.

Actions:

- Keep one clear H1 that closely matches the page's primary query.
- Put the differentiator in the title: "photo", "free", "AI", "macro", "iPhone and Android".
- Avoid titles that are too broad for the query. For example, "Calkilo | AI calorie tracker and nutrition app" is weaker for Italian users than "Calcolo calorie AI da foto | Calkilo".
- Make meta descriptions specific enough to preview the task users searched for, not just app branding.

### 4. Fix duplicate/legacy URL handling with HTTP redirects

Live checks on 2026-05-06:

- `https://calkilo.com/index.html` returns `200` with canonical `https://calkilo.com/`.
- `https://calkilo.com/contact` returns `200` with a meta refresh/noindex legacy page to `https://calkilo.com/contact/`.
- `https://calkilo.com/fa` correctly returns `301` to `https://calkilo.com/fa/`.

Canonical tags are useful, but they are hints. The cleaner fix is an HTTP redirect:

- `/index.html` -> `/`
- `/contact` or `/contact.html` -> `/contact/`
- `/faq.html` -> `/faq/`

Because the live headers show GitHub/Fastly/Cloudflare, this may need Cloudflare redirect rules rather than a Next.js-only change.

### 5. Review sitemap freshness and simplify ignored fields

The sitemap includes canonical URLs and resource pages, which is good. It also contains `changefreq` and `priority`, which Google says it does not use. More importantly, `lastmod` should only be kept where it reflects a significant page update.

Recommended:

- Keep canonical URLs in the sitemap.
- Add localized resource pages once they exist.
- Update `lastmod` only when main content, structured data, or important links change.
- Do not rely on `changefreq` or `priority` for crawling or ranking.

### 6. Treat FAQ rich results as a low-probability bonus

The site has visible FAQ content and FAQPage JSON-LD. That is structurally fine if validated, but Google currently limits FAQ rich results mainly to well-known authoritative government or health-focused sites. The `Search appearance.csv` export is empty, so there is no evidence that rich-result appearance is currently driving traffic.

Keep FAQ markup only where the FAQ is visible on the page and accurate. Do not expect FAQ structured data to be the main growth lever.

### 7. Improve mobile-first performance hygiene

Mobile is the strongest traffic source. The current homepage loads several visual assets well, but `components/LandingPage.tsx` marks all four AI screen images as `loading="eager"` even when only one is active. Those files are roughly:

- `analysis.png`: 132 KB
- `chatAi.png`: 152 KB
- `goal.png`: 80 KB
- `food.png`: 324 KB

Recommended:

- Keep the first active AI image eager/high priority only if it is visible above the fold on mobile.
- Lazy-load inactive AI images.
- Prefer WebP/AVIF for the PNG app screenshots if visual quality remains acceptable.
- Remove unused large PNG duplicates from `public/assets/figma/` if they are not referenced anywhere.

### 8. Strengthen AI-search readiness with useful, visible content

For AI features in Search, Google's guidance is still grounded in classic SEO: useful unique content, crawlable visible content, and structured data matching what users can see. For Calkilo, that means pages should explain the product task clearly rather than only repeating marketing claims.

Recommended content modules:

- "How Calkilo estimates calories from a food photo"
- "What to check before saving an AI estimate"
- "How macro tracking works after photo logging"
- "Privacy and food photo handling"
- "When a photo calorie estimate can be wrong"
- "Calkilo vs manual calorie logging"

These modules should be localized for Persian and Italian instead of translated only in navigation/footer.
